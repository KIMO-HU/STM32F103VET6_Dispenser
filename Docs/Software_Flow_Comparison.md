# 中药机终端软件流程对比分析

> 对比对象：旧版 RT-Thread 软件 (V1.0_20250806) vs 新版裸机软件 (V1.6.0)
> 生成日期: 2026-05-13
> 最近更新: 2026-05-13 (增加看门狗、料位LED指示对比)

---

## 1. 概述

本文档基于对 `Docs/KZB-ZYFYJ-HCX-V1.2_20260507/` 目录下旧版 RT-Thread 软件代码的完整分析，以及新版裸机软件代码的对比，整理出两份软件的详细流程图，并标注流程不一致的问题点。

---

## 2. 旧版软件（RT-Thread）流程图

### 2.1 系统架构

旧版软件基于 **RT-Thread RTOS**，采用多线程 + 定时器中断 + 消息队列的架构：

```
┌─────────────────────────────────────────────────────────────┐
│                      RT-Thread 内核                          │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │  uartsend   │  │  uartrecv   │  │   check_thread      │  │
│  │  线程       │  │  线程       │  │   (自检/心跳)       │  │
│  └──────┬──────┘  └──────┬──────┘  └─────────────────────┘  │
│         │                │                                    │
│  ┌──────▼────────────────▼──────┐  ┌─────────────────────┐  │
│  │      消息队列 / 信号量        │  │  DI_FilterThread    │  │
│  │   uart4send_msgqueue         │  │  (DI 滤波线程)      │  │
│  │   uart4_msgqueue             │  │                     │  │
│  └──────────────────────────────┘  └─────────────────────┘  │
│  ┌────────────────────────────────────────────────────────┐ │
│  │              TIM4 定时器中断 (1ms/2ms)                  │ │
│  │         HTTIMERExpiredISR() — 出药核心控制              │ │
│  └────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

**关键线程说明：**

| 线程 | 功能 | 优先级 |
|------|------|--------|
| `uartsend_thread` | UART 发送线程，从消息队列取数据发送 | 中等 |
| `uartrecv_thread` | UART 接收线程，接收数据写入消息队列 | 中等 |
| `check_thread` | 系统自检、心跳发送、LED 控制 | 低 |
| `DI_FilterThread` | 数字输入滤波（拨码开关等） | 低 |
| `TIM4_IRQHandler` | 定时器中断 — 出药状态机核心 | 中断 |

### 2.2 主程序初始化流程

```
main()
│
├─ 初始化 RT-Thread 内核
├─ 初始化硬件 (GPIO/UART/SPI/TIM/ADC/DMA)
├─ 初始化 ADXL 传感器、ADE9000 电能芯片 (注释掉，未实际使用)
├─ 初始化 IWDG 看门狗 (PR=64, RLR=1250, 超时~2秒)
├─ 从 EEPROM/Flash 加载参数 (SysParams)
├─ 创建并启动各线程
│   ├─ uartsend_thread (UART 发送)
│   ├─ uartrecv_thread (UART 接收)
│   ├─ check_thread (自检/心跳/料位LED)
│   └─ DI_FilterThread (DI 滤波)
├─ 初始化 TIM4 定时器 (出药用，默认关闭)
└─ 进入 RT-Thread 调度循环
```

### 2.3 通信协议处理流程

```
UART4 接收中断
│
└─> uartrecv_thread 读取数据
    │
    └─> 写入 uart4_msgqueue (消息队列)
        │
        └─> ProtocolSAC 解析 (在主线程或 check_thread 中)
            │
            ├─ 字节级状态机解析:
            │   HEAD1(0x55) → HEAD2(0xBB) → TYPE → REV1/2
            │   → LEN_HI/LO → DATA → CRC
            │
            └─ 根据 TYPE 分发处理:
                ├─ 0x01 IO 查询
                ├─ 0x04 单次出药 → 设置 MEDCINE_START_ORDER_FLAG=1
                ├─ 0x06 重量查询
                ├─ 0x09 药量设置
                ├─ 0x0F/0x20/0x21 高中低速设置
                ├─ 0x10 出药状态查询
                ├─ 0x19 手动开门 → 设置 DOOR_OPEN_ORDER_FLAG=1
                ├─ 0x1A 手动振动 → 设置 ACTION_VIBRATE_ORDER_FLAG=1
                ├─ 0x1C 高重标定 / 0x1D 零重标定
                ├─ 0x1E 绞龙时间 / 0x1F 绞龙重量阈值 / 0x22 低重量阈值
                ├─ 0x70~0x78 IAP 升级指令
                └─ 其他设置/查询指令...
                │
                └─> 构造响应帧 → 写入 uart4send_msgqueue
                    │
                    └─> uartsend_thread 发送
```

> **注意**：旧版协议 **不支持 0x61（急停）和 0x62（一键清空）** 命令。

### 2.4 出药控制核心 — MEDCINE_STEP 状态机

出药控制由 **两个并行的状态机** 协同完成：

#### 状态机 A: MEDCINE_STEP（出药速度/阶段控制）

```
┌─────────┐    Usv_MEDCINE_Start()      ┌─────────┐
│ STEP=0  │ ──────────────────────────> │ STEP=3  │  高速阶段
│ (空闲)  │   (单次出药 0x04 触发)      │ (高速)  │
└─────────┘                             └────┬────┘
     ▲                                       │
     │        重量 >= 75% 目标值              │ 重量 < 75%
     │        (applicationSAC.c 中判断)      │
     │                                       ▼
     │                                  ┌─────────┐
     │                                  │ STEP=2  │  中速阶段
     │                                  │ (中速)  │
     │                                  └────┬────┘
     │                                       │ 重量 >= 50% 且 < 75%
     │                                       │
     │                                       ▼
     │                                  ┌─────────┐
     │                                  │ STEP=1  │  低速阶段
     │                                  │ (低速)  │
     │                                  └────┬────┘
     │                                       │ 重量 >= 100%
     │                                       │ 且持续 10 秒
     │                                       ▼
     │                                  ┌─────────┐
     │                                  │ STEP=4  │  完成阶段
     │                                  │ (完成)  │
     │                                  └────┬────┘
     │                                       │ TIM4 中断检测到
     │                                       │ MEDCINE_STEP==4
     │                                       │ → 停止绞龙, 归零 DAC
     └───────────────────────────────────────┘
```

**MEDCINE_STEP 各阶段说明：**

| STEP | 含义 | 触发条件 | DAC 输出 |
|------|------|----------|----------|
| 0 | 空闲/停止 | 初始状态或出药结束 | 0 |
| 3 | 高速出药 | 单次出药命令触发 | SPEED_MODE_H (0x0FFF) |
| 2 | 中速出药 | 当前重量 ≥ 50% 目标值 | SPEED_MODE_M (0x07FF) |
| 1 | 低速出药 | 当前重量 ≥ 75% 目标值 | SPEED_MODE_L (0x04FF) |
| 4 | 出药完成 | 当前重量 ≥ 100% 目标值，持续 10 秒 | 0 (小绞龙低速收尾) |

#### 状态机 B: MOTOR_STEP（电机/门动作控制）

`MOTOR_STEP` 在 `HTTIMERExpiredISR()` (TIM4 中断，约 1ms) 中逐周期更新：

```
MOTOR_STEP 状态转换图:

┌─────────┐
│  STEP=8 │ ──(Usv_MEDCINE_Start 首次出药)──> MOTOR_STEP=1
│ (上电)  │

     │ 自动转换

└────┬────┘
     │
     │ 自动转换
     ▼
┌─────────┐     自动转换

└────┬────┘
     │
     │ 自动转换
     ▼
┌─────────┐
│ STEP=0  │  (空闲状态)
│ (空闲)  │
└─────────┘

其他 MOTOR_STEP 状态:
┌─────────┐
│ STEP=5  │  手动开门命令 → 开门反转

└────┬────┘

     ▼
```

### 2.5 速度切换逻辑（重量判断）

速度切换在 `applicationSAC.c` 的某个线程/循环中执行（非中断），基于 `WEIGHT_GRAMDATA2`（当前重量）：

```
if (WEIGHT_GRAMDATA2 >= DATA_MEDCINE) {
    // 当前重量 ≥ 100% 目标值
    if (MEDCINE_TIME >= 10000ms 且 持续 10 次) {
        MEDCINE_STEP = 4;       // 进入完成阶段
        DAC = 0x0333 (小绞龙低速收尾);
    }
}
else if (WEIGHT_GRAMDATA2 >= (3 * DATA_MEDCINE) / 4) {
    // 75% ≤ 当前重量 < 100%
    MEDCINE_STEP = 1;           // 低速
    DAC = SPEED_MODE_L;
}
else if (WEIGHT_GRAMDATA2 >= (1 * DATA_MEDCINE) / 2) {
    // 50% ≤ 当前重量 < 75%
    MEDCINE_STEP = 2;           // 中速
    DAC = SPEED_MODE_M;
}
else {
    // 当前重量 < 50%
    MEDCINE_STEP = 3;           // 高速
    DAC = SPEED_MODE_H;
}
```

**速度切换阈值：固定 50% / 75%**，不可通过协议配置。

### 2.6 超时检测逻辑

```
TIM4 中断 (HTTIMERExpiredISR)
│
├─ MEDCINE_TIME++ (每中断周期递增)
│
└─ if (MEDCINE_TIME >= TIMEOUT_TIME * 1000) {
       MEDCINE_START_ORDER_FLAG = 0;
       MEDCINE_TIMEOUT_FLAG = 1;
       MEDCINE_STEP = 0;
       // 停止绞龙, 归零 DAC
   }
```

### 2.7 防悬空电机控制

防悬空电机（DC1/DC2）在 **TIM4 中断** 中控制：

```
TIM4 中断中 (HTTIMERExpiredISR):
│
├─ if (MEDCINE_STEP == 1/2/3 出药阶段) {
│      // 每 5 个中断周期一个循环
│      if (TIM4TIMES % 5 == 0~4) {
│          // 周期性正转/停止交替
│          GPIO_SetBits(GPIOD, GPIO_Pin_6);   // 正转
│      }
│   }
│
└─ if (MEDCINE_STEP == 4 完成阶段 或 手动开门) {
       // 停止防悬空电机
       GPIO_ResetBits(GPIOD, GPIO_Pin_6);
   }
```

### 2.8 参数存储

```
参数修改 (protocolSAC.c 中处理设置指令)
│
└─> 修改 SysParams 结构体中的对应字段
    │
    └─> 调用 EEPROM 写入 (I2C 接口)
        │
        └─> 参数保存到外部 EEPROM
```

## 3. 新版软件（裸机）流程图

### 3.1 系统架构

新版软件采用 **裸机轮询架构**，无 RTOS，所有逻辑在主循环中顺序执行：

```
┌─────────────────────────────────────────────────────────────┐
│                    裸机轮询架构                               │
│                                                              │
│  ┌───────────────────────────────────────────────────────┐  │
│  │              main() while(1) 主循环                    │  │
│  │  ┌─────────────────────────────────────────────────┐  │  │
│  │  │ 1. UART_GetChar() → Protocol_ParseFrame()       │  │  │
│  │  │    (轮询接收, 字节级状态机解析)                   │  │  │
│  │  └─────────────────────────────────────────────────┘  │  │
│  │  ┌─────────────────────────────────────────────────┐  │  │
│  │  │ 2. DISPENSER_Process() 出药状态机               │  │  │
│  │  │    (重量读取 → 超时检测 → 速度切换 → 电机控制)  │  │  │
│  │  └─────────────────────────────────────────────────┘  │  │
│  │  ┌─────────────────────────────────────────────────┐  │  │
│  │  │ 3. LED 心跳控制 (sysTickCount % 1000)           │  │  │
│  │  └─────────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────────────┐  │
│  │        SysTick 中断 (1ms)                             │  │
│  │        sysTickCount++                                 │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 主程序初始化流程

```
main()
│
├─ SCB->VTOR = 0x08008000 (App 向量表偏移)
├─ SystemClock_Config()        // 72MHz
├─ SysTick_Init()              // 1ms 中断
├─ GPIO_Init_All()             // 全部 GPIO
├─ DAC7512_Init()              // 双路 DAC 模拟输出
├─ UART4_Init(115200)          // 主通信 RS485
├─ USART3_Init(115200)         // 备用 RS485 (用于料位LED指示)
├─ DISPENSER_Init()            // 初始化 ADS1256, 从 Flash 加载参数
├─ LED_Indicator_Init()        // 初始化料位LED指示器
├─ IWDG_Init()                 // 初始化看门狗 (PR=64, RLR=1250, ~2s)
├─ __enable_irq()
├─ LED 闪烁 3 次 (启动指示)
│
└─ while(1) 主循环
     ├─ UART_GetChar() → Protocol_ParseFrame()  [通信处理]
     ├─ DISPENSER_Process()                      [出药状态机]
     ├─ LED_Indicator_Update()                   [料位LED指示]
     ├─ LED 心跳                                  [状态指示]
     └─ IWDG_Feed()                               [喂看门狗]
```

### 3.3 通信协议处理流程

```
main() while(1)
│
├─ UART_GetChar(UART_IDX_UART4, &ch)
│   │
│   └─ 有数据 → Protocol_ParseFrame(&ch, 1)
│       │
│       └─ 字节级状态机:
│           ST_HEAD1(0x55) → ST_HEAD2(0xBB) → ST_TYPE → REV1/2
│           → ST_LEN_HI/LO → ST_DATA → ST_CRC
│           │
│           ├─ CRC 正确 → Protocol_ProcessCommand()
│           └─ CRC 错误 → 回到 ST_HEAD1
│               │
│               └─ 根据 cmd_type 分发:
│                   ├─ 0x01 IO 查询 → 读取 GPIO → 发送响应
│                   ├─ 0x04 单次出药 → g_disp_params.target_weight=weight; g_dispense_cmd=1
│                   ├─ 0x06 重量查询 → ADS1256_ReadWeight() → 发送响应
│                   ├─ 0x09/0x0F/0x20/0x21 参数设置 → DISPENSER_SetParam() → PARAMS_Save()
│                   ├─ 0x10 出药状态查询 → 发送 g_disp_status
│                   ├─ 0x19 手动开门 → g_door_open_cmd=1
│                   ├─ 0x1A 手动振动 → g_vibrate_cmd=1
│                   ├─ 0x1C 高重标定 / 0x1D 零重标定 → ADS1256 标定
│                   ├─ 0x61 急停 → g_emergency_stop=1
│                   ├─ 0x62 一键清空 → g_dispense_cmd=0xFF
│                   ├─ 0x70/0x71 IAP 升级 → IAP_TriggerBootloader()
│                   └─ 其他设置/查询指令...
│
├─ LED_Indicator_Update() [料位状态变化时发送LED颜色帧]
│   └─ 读取料位传感器 (低/中/高)
│   └─ 状态变化时通过 USART3 发送对应颜色帧
│       ├─ 全空 → 红色 (21字节帧)
│       ├─ 仅低位/仅中位/仅高位 → 33%蓝色 (114字节LED条帧)
│       ├─ 低位+中位/低位+高位/中位+高位 → 66%蓝色 (114字节LED条帧)
│       └─ 全满 → 浅蓝色 (21字节帧)
│                       │
│                       └─ Protocol_SendResponse() → UART_PutBuf()
│
└─ 无数据 → 跳过
```

### 3.4 出药控制核心 — DISPENSER_Process 状态机

```
DISPENSER_Process() (主循环每轮调用)
│
├─ 1. 急停检查 (最高优先级)
│   └─ if (g_emergency_stop) → DISPENSER_EmergencyStop()
│       ├─ 停止所有电机
│       ├─ DAC7512_SetZeroAll()
│       ├─ g_disp_status = DISP_IDLE
│       └─ g_emergency_stop = 0
│
├─ 2. 手动开门检查
│   └─ if (g_door_open_cmd) → DISPENSER_DoorOpen()
│       ├─ MOTOR_DC3_Set(1, 0) 开门
│       ├─ s_door_is_open = 1
│       └─ s_door_open_time = sysTickCount
│
├─ 3. 手动振动检查
│   └─ if (g_vibrate_cmd) → DISPENSER_Vibrate()
│       ├─ MOTOR_DC1_Set(1,0); MOTOR_DC2_Set(1,0)
│       ├─ delay_ms(500)
│       └─ MOTOR_DC1_Set(0,0); MOTOR_DC2_Set(0,0)
│
├─ 4. 一键清空检查
│   └─ if (g_dispense_cmd == 0xFF) → DISPENSER_Clear()
│       ├─ DISPENSER_Stop()
│       ├─ MOTOR_DC3_Set(1, 0) 开门
│       ├─ AUGER_B_Set(1, 1, 0) 大绞龙反转
│       └─ delay_ms(3000) → DISPENSER_Stop()
│
├─ 5. 出药启动检查
│   └─ if (g_dispense_cmd && g_disp_status == DISP_IDLE)
│       → DISPENSER_Start(g_disp_params.target_weight)
│       ├─ g_disp_status = DISP_HIGH_SPEED
│       ├─ s_dispense_start_time = sysTickCount
│       ├─ MOTOR_DC3_Set(1, 0) 开门
│       ├─ AUGER_S_Set(1,0,0); AUGER_B_Set(1,0,0)
│       └─ DAC7512_Write(DAC_CH_SMALL_AUGER, high_speed)
│          DAC7512_Write(DAC_CH_BIG_AUGER, high_speed)
│
├─ 6. 出药状态处理 (g_disp_status != IDLE)
│   │
│   ├─ 6.1 超时检测
│   │   └─ if (elapsed > timeout * 1000ms)
│   │       → DISPENSER_Stop(); g_disp_status = DISP_TIMEOUT
│   │       (TIMEOUT 状态需外部复位才能清除)
│   │
│   ├─ 6.2 重量读取
│   │   └─ g_disp_params.current_weight = ADS1256_ReadWeight()
│   │
│   ├─ 6.3 完成检测 — 已与旧版一致
│   │   └─ if (current_weight >= target_weight)
│   │       → s_complete_confirm_cnt++
│   │       → if (s_complete_confirm_cnt >= 10)  // 持续约10秒
│   │           → DISPENSER_Stop(); g_disp_status = DISP_IDLE
│   │       → else 继续出药 (小绞龙低速收尾)
│   │
│   ├─ 6.4 速度切换
│   │   └─ remaining = target_weight - current_weight
│   │       ├─ remaining ≤ low_weight_thresh
│   │       │   → DISP_LOW_SPEED
│   │       │   → 小绞龙低速, 大绞龙停止
│   │       │   → DAC7512_Write(SMALL, low_speed); DAC7512_SetZero(BIG)
│   │       ├─ remaining ≤ target * speed_percent / 100
│   │       │   → DISP_MID_SPEED
│   │       │   → 大小绞龙中速
│   │       │   → DAC7512_Write(SMALL, mid_speed); DAC7512_Write(BIG, mid_speed)
│   │       └─ else
│   │           → DISP_HIGH_SPEED
│   │           → 大小绞龙高速
│   │           → DAC7512_Write(SMALL, high_speed); DAC7512_Write(BIG, high_speed)
│   │
│   └─ 6.5 防悬空电机
│       └─ if (sysTickCount % 500 < 250)
│              MOTOR_DC1_Set(1,0); MOTOR_DC2_Set(1,0)
          else MOTOR_DC1_Set(0,0); MOTOR_DC2_Set(0,0)
│
└─ 7. 料斗门自动关闭
    └─ if (s_door_is_open && elapsed > door_time * 1000ms)
        → MOTOR_DC3_Set(0, 0); s_door_is_open = 0
```

### 3.5 速度切换逻辑（重量判断）—— 已与旧版一致

```
dispensed = current_weight
target = target_weight

if (dispensed >= target * 3 / 4) {
    // 已出重量 ≥ 75% 目标值 -> 低速 (对应旧版 MEDCINE_STEP=1)
    g_disp_status = DISP_LOW_SPEED;
}
else if (dispensed >= target / 2) {
    // 已出重量 ≥ 50% 目标值 -> 中速 (对应旧版 MEDCINE_STEP=2)
    g_disp_status = DISP_MID_SPEED;
}
else {
    // 已出重量 < 50% -> 高速 (对应旧版 MEDCINE_STEP=3)
    g_disp_status = DISP_HIGH_SPEED;
}
```

**速度切换阈值：固定 50% / 75%**（与旧版一致）

### 3.6 超时检测逻辑 — 已与旧版一致

```
elapsed = sysTickCount - s_dispense_start_time

if (elapsed > timeout * 1000ms) {
    DISPENSER_Stop();
    g_disp_status = DISP_IDLE;  // 与旧版一致: 超时后回到 IDLE
    return;
}
```

### 3.7 防悬空电机控制 — 已与旧版一致

防悬空电机在主循环中控制（周期约 5ms）：

```
if (sysTickCount % 5 < 3) {  // 3ms ON / 2ms OFF
    MOTOR_DC1_Set(1, 0);  // 正转
    MOTOR_DC2_Set(1, 0);
} else {
    MOTOR_DC1_Set(0, 0);  // 停止
    MOTOR_DC2_Set(0, 0);
}
```

## 4. 流程差异对比分析

### 4.1 架构差异

| 项目 | 旧版软件 | 新版软件 | 影响 |
|------|----------|----------|------|
| **操作系统** | RT-Thread RTOS (多线程) | 裸机轮询 | 新版无任务切换开销，但无优先级调度 |
| **线程数** | 4+ 线程 + 中断 | 无线程，仅主循环+SysTick | 新版逻辑更简单，但实时性依赖轮询频率 |
| **通信处理** | 中断接收 + 消息队列 + 发送线程 | 轮询接收 + 直接解析 | 新版在通信繁忙时可能阻塞出药控制 |
| **代码量** | ~182K 行 (含 RT-Thread) | ~2K 行 (裸机核心) | 新版更精简，易于维护 |

### 4.2 状态机差异

| 项目 | 旧版软件 | 新版软件 | 风险/说明 |
|------|----------|----------|-----------|
| **状态机数量** | 2 个并行状态机 (MEDCINE_STEP + MOTOR_STEP) | 1 个单一状态机 (g_disp_status) | 旧版更复杂但分工明确；新版简化但可能遗漏边界条件 |
| **状态数量** | MEDCINE_STEP: 5 个; MOTOR_STEP: 10+ 个 | DISPENSER: 5 个 | — |
| **状态含义** | STEP=3(高速), 2(中速), 1(低速), 4(完成) | HIGH_SPEED, MID_SPEED, LOW_SPEED, TIMEOUT, IDLE | 状态编码不一致 |
| **门控制状态** | MOTOR_STEP=1/2/3/4/10/11 专门管理门开关 | 简化：启动时开门，定时器自动关门 | 新版门控制更简单，但可能缺少旧版的门到位检测逻辑 |

### 4.3 速度切换逻辑差异 ⚠️

| 项目 | 旧版软件 | 新版软件 | 风险/说明 |
|------|----------|----------|-----------|
| **高速→中速阈值** | 固定 50% (当前重量) | 可配置 `speed_percent` (默认 50%，基于**剩余重量**) | ⚠️ **逻辑相反！** 旧版基于"已出重量"，新版基于"剩余重量" |
| **中速→低速阈值** | 固定 75% (当前重量) | 可配置 `low_weight_thresh` (默认 0，基于**剩余重量**) | ⚠️ **逻辑相反！** 且新版默认值 0 意味着可能不经过低速阶段 |
| **低速→完成条件** | 重量 ≥ 100% 持续 10 秒 | 重量 ≥ 目标值 (即时) | 新版缺少"持续确认"机制，可能因抖动提前结束 |
| **完成阶段处理** | MEDCINE_STEP=4，小绞龙低速收尾 | 直接停止，DAC 归零 | 旧版有收尾阶段，新版直接停止 |
| **速度参数默认值** | H=0x0FFF, M=0x07FF, L=0x04FF | H=0x06FF, M=0x05FF, L=0x04FF | 新版默认值更低，出药速度可能更慢 |

**⚠️ 关键不一致：速度切换方向相反**

旧版逻辑（基于已出重量百分比）：
```
已出重量 < 50%  → 高速
50% ≤ 已出重量 < 75% → 中速
75% ≤ 已出重量 < 100% → 低速
已出重量 ≥ 100% → 完成
```

新版逻辑（基于剩余重量）：
```
剩余重量 > target×50% → 高速
剩余重量 ≤ target×50% 且 > low_weight_thresh → 中速
剩余重量 ≤ low_weight_thresh → 低速
```

当 `speed_percent=50`, `low_weight_thresh=0` 时，新版逻辑等价于：
```
已出重量 < 50% → 高速
50% ≤ 已出重量 < 100% → 中速
已出重量 ≥ 100% → 低速 (但 low_weight_thresh=0 时不会进入)
```

这意味着新版默认配置下**没有低速阶段**，直接从高速→中速→完成。

### 4.4 命令支持差异

| 命令码 | 旧版 | 新版 | 说明 |
|--------|------|------|------|
| 0x04 单次出药 | ✅ | ✅ | — |
| 0x61 急停 | ❌ **不支持** | ✅ | 新版新增 |
| 0x62 一键清空 | ❌ **不支持** | ✅ | 新版新增 |
| 0x19 手动开门 | ✅ | ✅ | — |
| 0x1A 手动振动 | ✅ | ✅ | — |
| 0x1C 高重标定 | ✅ | ✅ | — |
| 0x1D 零重标定 | ✅ | ✅ | — |
| 0x70~0x78 IAP | ✅ | ✅ | — |

### 4.5 超时处理差异

| 项目 | 旧版软件 | 新版软件 | 风险/说明 |
|------|----------|----------|-----------|
| **超时后状态** | MEDCINE_STEP=0, 可立即重新出药 | g_disp_status=DISP_TIMEOUT, **需外部复位** | ⚠️ 新版超时后不能自动恢复，需要上位机发送指令清除 |
| **超时时间单位** | 秒 (TIMEOUT_TIME) | 秒 (g_disp_params.timeout) | 一致 |
| **超时动作** | 停止绞龙, DAC 归零 | 停止绞龙, DAC 归零 | 一致 |

### 4.6 防悬空电机控制差异

| 项目 | 旧版软件 | 新版软件 | 风险/说明 |
|------|----------|----------|-----------|
| **控制位置** | TIM4 中断 (1ms) | 主循环 (与 DISPENSER_Process 同周期) | 新版控制频率取决于主循环执行速度 |
| **周期** | 约 5ms (TIM4TIMES % 5) | 500ms (sysTickCount % 500) | ⚠️ 新版周期 100 倍于旧版，防悬空效果可能不同 |
| **动作模式** | 正转/停止交替 (脉冲式) | 正转/停止交替 (250ms ON / 250ms OFF) | — |

### 4.7 门控制差异

| 项目 | 旧版软件 | 新版软件 | 风险/说明 |
|------|----------|----------|-----------|
| **门状态检测** | DOOR_STAUTS_FLAG (基于传感器) | s_door_is_open (软件标志) | 新版缺少硬件门状态反馈 |
| **开门时机** | 出药开始时开门 (MOTOR_STEP=3) | 出药开始时开门 | 一致 |
| **关门时机** | 复杂的 MOTOR_STEP 状态机转换 | 定时器超时自动关门 | 新版简化，但可能缺少到位检测 |
| **关门动作** | 专门的关门电机状态 (MOTOR_STEP=1/2) | DC3 电机直接控制 | — |

### 4.8 重量采集差异

| 项目 | 旧版软件 | 新版软件 | 风险/说明 |
|------|----------|----------|-----------|
| **ADC 类型** | 可能使用 STM32 内部 ADC + DMA | ADS1256 24bit Σ-Δ ADC | 新版精度更高 (24bit vs 12bit) |
| **重量变量** | WEIGHT_GRAMDATA, WEIGHT_GRAMDATA1/2/3/4 (多路) | g_disp_params.current_weight (单路) | 旧版有多路重量，新版仅单路 |
| **滤波算法** | 复杂的均值/冒泡滤波 (20 点采样) | 直接读取 ADS1256 输出 | 新版缺少软件滤波，可能受噪声影响 |
| **标定存储** | EEPROM | 内部 Flash | — |

### 4.9 看门狗差异

| 项目 | 旧版软件 | 新版软件 | 风险/说明 |
|------|----------|----------|-----------|
| **看门狗类型** | IWDG (独立看门狗) | IWDG (独立看门狗) | 一致 |
| **预分频器** | /64 | /64 | 一致 |
| **重装载值** | 1250 | 1250 | 一致 |
| **超时时间** | ~2秒 | ~2秒 | 一致 |
| **初始化位置** | board.c → main() 中调用 | watchdog.c → main() 中调用 | 新版独立模块，寄存器级实现 |
| **喂狗位置** | 各线程/中断中分散调用 | 主循环末尾统一调用 | 新版统一在主循环喂狗，更简洁 |
| **关键操作喂狗** | configSAC.c/ymodem.c中大量调用 | 当前仅在主循环喂狗 | ⚠️ 如需在Flash擦写等长操作中保护，应在操作期间额外喂狗 |

### 4.10 料位LED指示差异

## 6. 建议修改清单

### 已完成的修改 ✅

1. ✅ **统一速度切换逻辑**：已改为基于"已出重量"百分比（<50%高速 / 50%-75%中速 / 75%-100%低速）。
2. ✅ **修复防悬空电机周期**：已改为 5ms 周期（`sysTickCount % 5`）。
3. ✅ **确认并修复硬件引脚**：小绞龙 EN 改为 PA10，大绞龙 FR 改为 PD2。
4. ✅ **增加完成确认延时**：重量≥目标值需持续 10 秒才确认完成。
5. ✅ **超时自动恢复**：超时后状态自动回到 IDLE。
6. ✅ **增加料位LED指示**：通过USART3发送协议帧控制外部LED颜色，与旧版逻辑一致。
7. ✅ **增加看门狗保护**：IWDG配置与旧版一致(PR=64,RLR=1250,~2s)。

### 建议优化（后续迭代）

8. ✅ **增加重量读数软件滤波（滑动平均）**：在 `ads1256.c` 中增加 8 点滑动平均滤波，`ADS1256_ReadWeight()` 返回滤波后的重量值。
9. ✅ **增加门到位传感器检测**：`dispenser.c` 中门关闭时预留了 `SENSOR_ReadDoor()` 检测接口（当前仍以时间为主，硬件确认可扩展）。
10. ✅ **增加二次出药/续药逻辑**：超时后如果未达到目标重量（差值 > `auger_weight_diff`），自动标记 `s_timeout_need_resume`，在下次主循环中自动重新启动出药。
11. ✅ **看门狗在Flash擦写等长操作中增加额外喂狗保护**：`params_storage.c` 中 `flash_erase_page()` 在擦除前后各调用一次 `IWDG_Feed()`。
